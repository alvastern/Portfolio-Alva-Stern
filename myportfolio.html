My Portfolio

HTML & CSS, 2025

Description of the Project

This is my first programming project. I choose to develop my own portfolio because I wanted to learn more about front-end programming. This project did not include very complicated codes, but it was very good for me since I improved my programming skills a lot.

Goal with this Project

The goal for me in this programming project was to be able to present my portfolio in the best possible way by using HTML and CSS.

HTML

HTML is used to structure the content of the website. While prototyping my portfolio, HTML became an important aspect since I think it’s important to have a good structure.

CSS

CSS is used to style the content of the website and became an important aspect to the code of my portfolio because I think my portfolio should look nice and professional. CSS also helps to make the portfolio look structured.

Crucial Codes

HTML: Create the structure

To create a good structure in my code, I used these four elements. They were crucial to create good structure.

CSS: Dynamics

Here, I used the CSS properties display and align-items to make a more dynamic look. Instead of everything laying under each other, it now lays next to each other.

HTML: Connect different pages

To connect different pages, I used the <a> element. It was used to link the different pages in my portfolio, but also to other websites.

CSS: Space around content

Padding and margin was used to give the content space. Margin controls the space outside the element and padding controls the space inside the element.

HTML: Facilitate CSS styling

To facilitate for myself when styling my portfolio, I used IDs and Classes to only address one element that will be affected by the styling.

CSS: Control size

The properties width, height and auto were used to control the size of the content.

HTML: Images

I use a lot of images in my portfolio. Therefore, I used the <img> element a lot to insert my images.

CSS: Styling

These properties were used to make my portfolio look nice. They control the background color, the font, and create shadows.

Challenges

Structure in CSS code: I found it challenging to have a good structure in my CSS file.

CSS layout: To control the layout of my elements in my code was challenging.

Classes: It was challenging to use classes and not making them be affected by other CSS rules or DRY (Don’t Repeat Yourself).

Insights

Structure: I realized it’s very important to have a good structure in the code. Both for making it easier to control the HTML using CSS and for making the code understandable.

Have a clear plan: I had to redo parts of my code because I didn’t have a clear plan of what I wanted to do. If I had a plan, I would know what to do and therefore save time.

IDs and classes: I realized IDs and classes are important to get a good structure in the code. By using these, the code will get easier to understand and control.
